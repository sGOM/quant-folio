# 07. 용어 사전 — 퀀트 · 증권 도메인

개발은 익숙해도 증권 도메인이 낯설면 코드의 "왜" 가 안 보인다. 코드에 자주 나오는
용어만 추렸다. (전략별 상세 수식은 `docs/strategies.md`)

---

## 거래 기본

| 용어 | 뜻 | 코드 등장 |
|------|-----|-----------|
| **종목코드(symbol)** | 6자리 KRX 코드. 예: 삼성전자 `005930` | 거의 모든 곳 |
| **현재가(quote)** | 지금 거래되는 가격 | `Quote`, `get_quote` |
| **호가** | 사자/팔자 대기 가격 | (이 프로젝트는 시장가 위주) |
| **시장가(market)** | 가격 안 정하고 "지금 즉시 체결" 주문 | `place_order(order_type="market")` |
| **지정가(limit)** | "이 가격 이하/이상일 때만" 주문 | `ORD_DVSN "00"` |
| **매수(buy)/매도(sell)** | 사기/팔기 | `OrderSide` |
| **체결(execution/fill)** | 주문이 실제로 거래 성사됨 | `Execution`, `Fill` |
| **미체결** | 접수됐지만 아직 안 팔림/안 사짐 | `OrderStatus.SUBMITTED` |
| **포지션(position)** | 현재 보유 중인 종목·수량·평균단가 | `Position` |
| **평균단가(avg_price)** | 보유분의 매입 평균 가격 | `Position.avg_price` |
| **OHLCV** | 시(Open)·고(High)·저(Low)·종(Close)·거래량(Volume) — 하루 봉의 5요소 | `price_ticks` |
| **일봉** | 하루를 한 칸으로 본 캔들. 이 프로젝트는 일봉 기준 | `load_ohlcv` |

---

## 주문 상태 (`OrderStatus`)

```
PENDING   엔진이 주문 생성, 아직 KIS 전송 전
SUBMITTED KIS 가 접수함(아직 체결 전일 수 있음)
PARTIAL   일부만 체결
FILLED    전량 체결
CANCELLED 취소됨
REJECTED  거부됨(잔고부족·장종료 등)
```

## 성과 지표 (백테스트 결과)

| 지표 | 뜻 | 직관 |
|------|-----|------|
| **수익률(total_return)** | 기간 총 손익률 | 높을수록 좋음 |
| **MDD(최대낙폭)** | 고점 대비 가장 크게 빠진 비율 | **낮을수록** 좋음(고통의 크기) |
| **샤프지수(Sharpe)** | 위험(변동성) 1단위당 초과수익 | 높을수록 효율적. 보통 1↑ 양호 |
| **승률(win_rate)** | 이긴 매매 비율 | 참고용(승률 높아도 손실 클 수 있음) |
| **자산곡선(equity curve)** | 시간에 따른 계좌 평가액 추이 | 우상향이 이상적 |

> **MDD 가 왜 중요한가**: 수익률이 같아도 MDD 가 크면 중간에 멘탈/자금이 못 버텨
> 전략을 포기하게 된다. 그래서 수익률만큼 낙폭을 본다.

## 리스크 관리 (`risk_limits`, 전략 config)

| 용어 | 뜻 | 코드 |
|------|-----|------|
| **손절(stop loss)** | 손실이 일정%면 강제 매도해 더 큰 손실 차단 | `stop_loss_pct`, `check_stop_loss` |
| **익절(take profit)** | 이익이 일정%면 매도해 수익 확정 | `take_profit_pct` |
| **트레일링 스탑** | 보유 중 **고점 대비** 일정% 하락 시 매도(이익을 따라 올림) | `trailing_stop_pct`, `trail:` 캐시 |
| **최대 포지션(max_position_size)** | 한 종목/전략에 넣을 최대 금액 | `evaluate_buy` |
| **일일 손실 한도(daily_loss_limit)** | 하루 손실이 이만큼이면 신규 진입 중단 | `check_daily_loss_limit` |

## 기술적 지표 (신호에 쓰임)

| 지표 | 한 줄 설명 | 신호 |
|------|------------|------|
| **SMA/EMA** | 단순/지수 이동평균(가격을 매끄럽게) | 골든크로스(단기↑장기)=매수 |
| **RSI** | 0~100, 과매수(70↑)/과매도(30↓) 측정 | 과매도 탈출=매수 |
| **MACD** | 두 EMA 의 차이로 추세·모멘텀 | 시그널선 상향돌파=매수 |
| **볼린저밴드** | 이동평균 ± 표준편차 밴드 | 하단 복귀=매수(평균회귀) |
| **ATR** | 평균 진폭(변동성 크기) | 트레일링/채널 폭 산정 |
| **돌파(breakout/Donchian)** | 최근 N봉 최고가 돌파 | 돌파=매수(추세추종) |
| **모멘텀** | 과거 대비 가격 추세 | 양전환=매수 |
| **스토캐스틱** | 고저 범위 내 현재가 위치(%) | 과매도 교차=매수 |
| **OBV** | 거래량으로 매집/분산 추적 | 추세 확인 |

> **평균회귀 vs 추세추종**: 볼린저·RSI·z-score 는 "빠지면 돌아온다"(평균회귀),
> 돌파·모멘텀·ATR트레일링은 "오르면 더 간다"(추세추종). 정반대 철학이라 시장 국면에
> 따라 잘 맞는 게 다르다.

## KIS API 용어

| 용어 | 뜻 |
|------|-----|
| **App Key / App Secret** | KIS 발급 API 자격증명(= Spring 의 client_id/secret) |
| **접근토큰(access token)** | 발급 후 24시간 유효. 분당 1회 발급 제한 → Redis 캐싱 |
| **approval_key** | 실시간(WebSocket) 시세 인증용 키 |
| **tr_id** | API 거래 식별 코드. 모의(`VTTC...`)/실전(`TTTC...`)이 다름 |
| **hashkey** | 주문 body 변조 방지 해시 |
| **모의투자(vts)** | 실거래와 똑같은 인터페이스의 **가짜 돈** 환경. 기본값 |
| **CANO / ACNT_PRDT_CD** | 계좌번호 앞 8자리 / 상품코드 2자리(`50012345-01`) |

## 멱등성 & 동시성 (개발 용어지만 이 프로젝트 맥락)

| 용어 | 이 프로젝트에서 | 
|------|------------------|
| **멱등성(idempotency)** | 같은 신호로 몇 번 시도해도 주문은 한 번만 — 이중 매수 방지 |
| **멱등성 키** | `s{전략}:{종목}:{side}:{신호봉날짜}` 결정적 문자열 |
| **분산 락(SET NX)** | 여러 프로세스/코루틴 중 하나만 임계영역 진입 |
| **TOCTOU** | "확인하고-쓰는" 사이 경합. 포지션 락으로 직렬화해 방지 |
| **슬라이딩 만료** | 활동 있으면 TTL 갱신(세션·가격캐시) |
| **하트비트** | 살아있음 신호. 엔진이 주기적으로 Redis 키 갱신 |
