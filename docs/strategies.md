# QuantFolio 전략 가이드 (strategies.md)

> 국내 주식(KRX) 퀀트 백테스팅·자동매매 플랫폼의 전략 카탈로그·타당성 검증·신규 전략 제안 문서
> 작성 기준 코드: `backend/app/services/backtest/signals.py`, `backend/app/services/backtest/engine.py`, `backend/engine/risk.py`
> 최종 갱신: 2026-06-22

이 문서는 **금융학적 타당성 검증** 관점에서 작성되었습니다. 신호 수식은 단일 출처인 `signals.py` 구현을 그대로 반영했으며, 추측이 아닌 코드 근거에 기반합니다. 백테스트 수치는 실거래를 보장하지 않습니다.

---

## 0. 공통 신호 규약 (구현 사실)

모든 전략은 `signals.py`에서 다음 규약을 따릅니다. 이 규약을 먼저 이해해야 각 전략의 한계를 정확히 판단할 수 있습니다.

- **신호는 "크로스오버 이벤트"**: 조건이 `거짓→참`으로 바뀌는 봉에서만 신호가 발생합니다(`_cross_up`/`_cross_down`). 연속 봉에서 동일 신호가 중복 발생하지 않습니다. 즉 진입 후 청산 신호가 나오기 전까지는 추가 진입 신호가 없습니다.
- **입력 시계열**: 기본 전략(SMA/EMA/RSI/MACD/볼린저/돌파/모멘텀/z-score/이격도)은 종가(close) 단일 시계열을 입력으로 받습니다. **OHLC 전략(atr_trailing·volatility_breakout·keltner·stochastic)** 은 OHLCV DataFrame 을 받습니다. `generate_signals(data, config)` 는 `data` 가 Series(종가)면 close-only 빌더로, DataFrame 이면 OHLC 빌더로 분기하며, close-only 빌더는 DataFrame 을 받아도 `close` 컬럼만 사용해 하위호환을 유지합니다(`requires_ohlc(config)` 로 입력 유형을 판별, 데이터 로더는 `get_close_series`/`get_ohlcv_frame` 로 공급).
- **체결 시점 가정**: 엔진(`engine.py`)은 vectorbt `Portfolio.from_signals`를 쓰며, 신호가 난 봉의 **해당일 종가(close)** 로 체결한다고 가정합니다(마커도 `close.loc[ts]`). 이는 미래참조 관점에서 주의가 필요합니다(2장 참조).
- **EMA는 `adjust=False`(재귀식)**: 실시간 엔진과 일관성을 위해 표준 재귀 EMA를 사용합니다. 올바른 선택입니다.
- **RSI는 Wilder 평활**(`ewm(alpha=1/period)`)을 사용합니다. 표준과 일치합니다.

## 1. 기존 전략 정리 (6종)

### 1.1 SMA Crossover (단순이동평균 교차) — `sma_crossover`

**개요**: 단기 단순이동평균(SMA)이 장기 SMA를 상향/하향 돌파할 때 매수/매도하는 고전적 추세추종 전략.

**신호 수식** (구현: `_ma_crossover_signals`, `_sma`)
- `SMA_n(t) = mean(close[t-n+1 .. t])` (`min_periods=n`, 미만 구간 NaN)
- 진입(매수): `SMA_fast(t) > SMA_slow(t)` AND `SMA_fast(t-1) <= SMA_slow(t-1)` (골든크로스)
- 청산(매도): `SMA_fast(t) < SMA_slow(t)` AND `SMA_fast(t-1) >= SMA_slow(t-1)` (데드크로스)
- 제약: `fast < slow` (위반 시 `ValueError`)

**금융학적 근거**: 가격에 추세(자기상관)가 존재한다는 가정. 이동평균은 노이즈를 평활해 추세 방향을 식별한다. 추세추종의 가장 기본형.

**파라미터**: `fast`(기본 5), `slow`(기본 20).

**적합 국면**: 명확한 방향성 추세장. 강한 상승/하락 추세에서 우수.

**주의·한계**:
- **횡보장 휩쏘(whipsaw)**: 박스권에서 잦은 교차로 거래비용만 누적, 연속 소액 손실.
- **지연(lag)**: 이동평균은 후행 지표라 추세 초반·종반을 놓침.
- SMA는 EMA보다 더 느리게 반응(과거 가중 동일).

**KRX 고려사항**: 골든크로스가 거의 매수신호로 알려져 군집 매매 → 슬리피지·체결지연 유발. 종가 체결 가정이 비현실적일 수 있음(2장).

---

### 1.2 EMA Crossover (지수이동평균 교차) — `ema_crossover`

**개요**: SMA Crossover와 동일 구조이나 지수이동평균(EMA) 사용. 최근 가격에 더 큰 가중치.

**신호 수식** (구현: `_ema`, `adjust=False`)
- `EMA_n(t) = α·close(t) + (1-α)·EMA_n(t-1)`, `α = 2/(n+1)`
- 진입/청산: SMA Crossover와 동일한 골든/데드크로스 (대상이 EMA)
- 제약: `fast < slow`

**금융학적 근거**: SMA와 동일하나 반응성이 높아 추세 전환을 더 빨리 포착. 대신 노이즈 민감도 증가.

**파라미터**: `fast`(기본 12), `slow`(기본 26).

**적합 국면**: 추세장. SMA보다 빠른 진입을 원할 때.

**주의·한계**: 반응성↑ → 횡보장 휩쏘가 SMA보다 더 잦을 수 있음. 노이즈 거짓신호 증가.

**KRX 고려사항**: SMA와 동일. 12/26은 MACD의 기본값과 같아 MACD와 신호가 유사하게 움직일 수 있음(전략 분산 효과 제한).

---

### 1.3 RSI (상대강도지수, Wilder) — `rsi`

**개요**: RSI가 과매도(lower) 영역에서 상향 복귀하면 매수, 과매수(upper) 영역에서 하향 이탈하면 매도하는 **평균회귀(역추세)** 전략.

**신호 수식** (구현: `_rsi`, `_rsi_signals`)
- `delta = close.diff()`, `gain = max(delta,0)`, `loss = max(-delta,0)`
- `AvgGain = Wilder_EMA(gain, alpha=1/period)`, `AvgLoss = Wilder_EMA(loss, alpha=1/period)`
- `RS = AvgGain / AvgLoss`, `RSI = 100 - 100/(1+RS)`
- 경계 처리: `AvgLoss=0 → RSI=100`, `AvgGain=AvgLoss=0 → RSI=50`
- 진입(매수): `RSI(t) > lower` AND `RSI(t-1) <= lower` (과매도 상향 복귀)
- 청산(매도): `RSI(t) < upper` AND `RSI(t-1) >= upper` (과매수 하향 이탈)
- 제약: `lower < upper`

**금융학적 근거**: 단기 과매도/과매수 후 평균 회귀 경향. Wilder(1978) 표준 정의.

**파라미터**: `period`(기본 14), `lower`(기본 30), `upper`(기본 70).

**적합 국면**: **횡보·박스권**. 추세 없는 구간에서 우수.

**주의·한계**:
- **강한 추세장에서 치명적**: 상승추세에서 RSI가 70 위에 장기 체류 → 매도 후 추가 상승 놓침. 하락추세에서 30 아래 체류 → 떨어지는 칼 매수.
- 진입은 "30 상향 복귀", 청산은 "70 하향 이탈"로 설계되어 진입과 청산 사이에 비대칭이 있음. 진입했으나 RSI가 70까지 못 가면 청산 신호 부재 → 리스크 스탑 의존.

**KRX 고려사항**: 상·하한가 ±30% 도달 종목은 RSI가 극단값에 고정될 수 있음. 평균회귀 가정이 깨지는 급등락 종목 주의.

### 1.4 MACD — `macd`

**개요**: MACD 선이 시그널 선을 교차할 때 매매하는 추세추종/모멘텀 전략.

**신호 수식** (구현: `_macd`)
- `MACD = EMA_fast(close) - EMA_slow(close)` (EMA는 `adjust=False`)
- `Signal = EMA_signal(MACD)`
- 진입(매수): `MACD(t) > Signal(t)` AND `MACD(t-1) <= Signal(t-1)`
- 청산(매도): `MACD(t) < Signal(t)` AND `MACD(t-1) >= Signal(t-1)`
- 제약: `fast < slow`

**금융학적 근거**: 두 EMA의 모멘텀 차이로 추세 강도·전환을 포착. 시그널선 교차로 진입 타이밍 결정. Appel 표준.

**파라미터**: `fast`(12), `slow`(26), `signal`(9).

**적합 국면**: 추세장·모멘텀 구간.

**주의·한계**:
- 히스토그램(MACD−Signal)이나 0선 교차는 사용하지 않음(시그널 교차만).
- 횡보장 휩쏘. 후행성.
- 12/26 EMA를 쓰므로 EMA Crossover와 신호 상관 높음.

**KRX 고려사항**: 종가 기준이라 데일리 봉에서 신호 지연. 분봉 운용 시 파라미터 재검토 필요.

### 1.5 Bollinger Bands (볼린저밴드) — `bollinger`

**개요**: 종가가 하단밴드를 상향 복귀하면 매수, 상단밴드를 하향 이탈하면 매도하는 **평균회귀** 전략.

**신호 수식** (구현: `_bollinger_signals`)
- `Mid = SMA(close, period)`
- `σ = rolling_std(close, period)` (pandas 기본 = 표본표준편차, ddof=1)
- `Upper = Mid + num_std·σ`, `Lower = Mid - num_std·σ`
- 진입(매수): `close(t) > Lower(t)` AND `close(t-1) <= Lower(t-1)`
- 청산(매도): `close(t) < Upper(t)` AND `close(t-1) >= Upper(t-1)`
- 제약 없음(period, num_std 양수 가정)

**금융학적 근거**: 가격이 ±2σ 밴드 내에서 평균회귀한다는 가정. Bollinger 표준.

**파라미터**: `period`(20), `num_std`(2.0).

**주의·한계**:
- **밴드워크(band walking)**: 강추세에서 가격이 밴드를 타고 지속 이동 → 평균회귀 신호가 역방향. 추세장 손실.
- 진입(하단 복귀)과 청산(상단 이탈)이 비대칭. 진입 후 상단밴드까지 못 가면 청산 부재 → 스탑 의존.
- `std`가 표본표준편차(ddof=1)임에 유의 — 모표준편차(ddof=0)를 쓰는 일부 도구와 미세 차이.

**KRX 고려사항**: 변동성 확장 시 밴드가 급격히 벌어져 신호 빈도 변동. 저유동성 종목은 σ 추정 불안정.

---

### 1.6 Breakout / Donchian (돌파) — `breakout`

**개요**: 직전 N봉 최고가를 상향 돌파하면 매수, 최저가를 하향 돌파하면 매도하는 추세추종(돌파) 전략.

**신호 수식** (구현: `_breakout_signals`)
- `Upper = rolling_max(close, period).shift(1)` (당일 제외)
- `Lower = rolling_min(close, period).shift(1)` (당일 제외)
- 진입(매수): `close(t) > Upper(t)` AND `close(t-1) <= Upper(t)`
- 청산(매도): `close(t) < Lower(t)` AND `close(t-1) >= Lower(t)`

**금융학적 근거**: 신고가/신저가 돌파는 추세 개시 신호(터틀 트레이딩 계열). 채널 돌파 = 모멘텀.

**파라미터**: `period`(기본 20).

**적합 국면**: 추세 개시·확장 구간. 변동성 확대 국면.

**주의·한계**:
- **종가 기준 채널** — 표준 Donchian은 고가/저가(high/low)로 채널을 만듭니다. 현재 구현은 **종가의 rolling max/min**이라 장중 돌파를 놓치고, 진폭이 과소평가됩니다(2장 권고).
- **`shift(1)`로 미래참조를 올바르게 차단**한 점은 우수 — 채널 계산에 당일 종가 미포함.
- 거짓 돌파(false breakout) 빈발. 돌파 후 되돌림.

**KRX 고려사항**: 상한가 직행 종목은 종가만으로는 돌파 강도 식별 불가. 갭 상승 시 종가 체결 가정이 비현실적.

## 2. 기존 구현의 금융학적 타당성 검증

심각도: **치명 / 높음 / 중간 / 낮음**

### [치명] 거래비용·세금·슬리피지 과소반영 — 백테스트 수익 과대평가

**문제**: `engine.py`는 `fees = config.get("fees", 0.00015)`(기본 0.015%) **단일 비율**만 사용합니다(`Portfolio.from_signals(..., fees=fees)`). 이 한 값으로 매수·매도를 동일 처리합니다.

**근거(시장 규칙)**:
- **증권거래세는 매도 시에만** 부과되며, **2026년 기준 코스피·코스닥 모두 0.20%** (코스피=거래세 0.05%+농특세 0.15%, 코스닥=거래세 0.20%)로 **인상**되었습니다. 0.18%(2025년)가 아닙니다.
- 증권사 위탁수수료(편도 약 0.01~0.015%)와 유관기관 수수료가 별도.
- 따라서 **왕복 실효 비용 ≈ 매수수수료(약 0.015%) + 매도수수료(약 0.015%) + 증권거래세 0.20% ≈ 0.23% 내외**. 현재 기본값 0.015%(편도)는 실제의 1/10 수준으로 비용을 과소평가.
- 슬리피지는 전혀 반영되지 않음(체결가 = 종가 가정).

**영향**: 회전율 높은 전략(RSI, 볼린저, 단기 교차)일수록 수익이 심각하게 과대평가. 거래가 잦을수록 백테스트 우위가 실거래에서 소멸.

**권고**:
1. 매도 측에 **증권거래세 0.20%를 별도 가산**. vectorbt 단일 `fees`로는 매수/매도 비대칭을 표현 못 하므로, 매도 신호에 추가 비용을 모델링하거나 보수적으로 왕복비용을 `fees`에 합산(예: 편도 환산 시 `fees ≈ 0.0012` 이상으로 상향).
2. **슬리피지 모델 추가**: 최소 호가단위 1틱, 또는 0.1~0.3% 고정 슬리피지(`Portfolio.from_signals`의 `slippage` 인자).
3. 기본값 `0.00015`를 비현실적 하한으로 인지하고 UI에서 경고.

### [높음] 종가 신호·종가 체결의 미래참조/실현가능성 문제

**문제**: 신호는 당일 종가로 계산되고(예: SMA/RSI/볼린저/MACD가 `close(t)` 포함), 엔진은 같은 봉의 **당일 종가로 체결**한다고 가정합니다(마커 `price = close.loc[ts]`).

**근거**: 종가는 장 마감(15:30) 시점 값입니다. "당일 종가 기반 신호를 당일 종가에 체결"하는 것은, 종가가 확정되는 순간에 그 종가로 주문을 넣는다는 의미로 **실거래에서 재현 불가**합니다(종가 단일가는 15:20~15:30 동시호가). 실시간 엔진은 종가 확정 후 신호를 보고 **익일 시가**에 체결하는 것이 현실적입니다.

**영향**: 백테스트가 약한 형태의 미래참조(체결가 낙관)를 내포. breakout만 `shift(1)`로 채널 미래참조를 차단했고 나머지는 당일 종가를 신호에 직접 사용.

**권고**:
- 표준화: **신호는 `close(t)` 기준, 체결은 `t+1` 시가(또는 t+1 종가)** 로 1봉 지연 체결을 도입(엔진에서 entries/exits를 `shift(1)`). 백테스트와 실시간 엔진의 체결 가정을 일치시켜야 함(이 부분은 quant-engine 에이전트 구현 영역).
- 적어도 백테스트 가정(당일 종가 체결)이 낙관적임을 결과 화면에 명시.

### [높음] KRX 호가단위·체결가 반올림 미반영

**문제**: 체결가가 연속 실수(종가) 그대로 사용됩니다. 실제 KRX는 가격대별 호가단위로 가격이 이산화됩니다.

**근거(2023.1.25 개편 후 현행 호가단위, 코스피·코스닥 공통)**:

| 주가 구간 | 호가단위 |
|---|---|
| ~2,000원 | 1원 |
| 2,000~5,000원 | 5원 |
| 5,000~10,000원 | 10원 |
| 10,000~20,000원 | 10원 |
| 20,000~50,000원 | 50원 |
| 50,000~100,000원 | 100원 |
| 100,000~200,000원 | 100원 |
| 200,000~500,000원 | 500원 |
| 500,000원~ | 1,000원 |

**영향**: 저가주는 호가단위 비율이 커서(예: 4,000원에서 5원 = 0.125%) 비용·수익 추정 오차가 거래세에 필적. 또한 채널/밴드 돌파가 호가단위 미만 차이로 발생하는 비현실적 신호 가능.

**권고**: 체결가를 해당 구간 호가단위로 반올림. 슬리피지를 "N틱"으로 모델링하면 호가단위와 자연스럽게 통합.

### [중간] 상·하한가(±30%) 미반영

**문제**: 상한가/하한가 도달 시 매수·매도 불가(체결 불가)이나 백테스트는 항상 체결을 가정.

**근거**: KRX 가격제한폭 ±30%. 상한가에서는 매도만, 하한가에서는 매수만 가능(사실상 미체결).

**영향**: 돌파·모멘텀 전략이 상한가 종목을 종가 체결한다고 가정하면 비현실적 수익. 손절도 하한가에서는 미체결.

**권고**: 일중 ±30% 도달 봉은 체결 제한 플래그 처리(데이터에 high/low 필요). 최소한 한계로 명시.

### [중간] 리스크 레이어의 백테스트/실거래 비대칭

**문제**:
- 백테스트(`engine.py`): 손절·익절·트레일링을 vectorbt의 `sl_stop`/`tp_stop`/`sl_trail`로 처리. 이들은 **종가 기준**으로 평가되며 장중 저가 터치를 반영하지 않음(close-only 가정).
- 실거래(`risk.py`): `check_stop_loss`는 `avg_price` 대비 `current_price` 하락률로 판단, `evaluate_buy`는 `max_position_size`, `check_daily_loss_limit`은 실현+평가손익으로 일일 한도 판단. **포지션 사이징은 "가용현금 전액 매수"**(`qty = cash // price`)에 가깝고 변동성 기반 사이징이 없음.

**영향**: 백테스트 손절은 장중 급락(저가)에서 트리거되어야 할 것이 종가 기준이라 손실을 과소평가. 실거래는 전액 매수로 단일 종목 집중 리스크.

**권고**:
1. 백테스트 손절을 **장중 저가(low) 기준**으로 평가하려면 OHLC 필요. 현재 close-only는 손절 손실을 낙관함을 명시.
2. 포지션 사이징을 변동성 타게팅/고정비율(예: 종목당 자본의 N%)로 개선(4장).
3. 일일 손실 한도 도달 시 신규 진입만 차단하고 기존 포지션 청산 정책은 별도 정의 필요.

### [중간] 과최적화 위험 — 표본 외 검증 부재

**문제**: 파라미터(fast/slow/period/lower/upper 등)가 자유롭게 조정 가능하나, 워크포워드/표본 외 검증 절차가 코드·프로세스에 없음.

**영향**: 특정 종목·기간에 맞춘 파라미터가 미래에 붕괴. 6개 전략 모두 소수 파라미터 의존.

**권고**: 4장 공통 권고의 워크포워드 분석 적용. 파라미터 민감도(인접 파라미터 성과 안정성) 점검.

### [낮음] 성과지표 정의·해석

**문제**: `sharpe_ratio()`가 vectorbt 기본(무위험수익률 0, freq="1D" 연율화). 소르티노·CAGR은 미산출. `win_rate`만 표시되면 손익비를 가려 오해 소지.

**권고**: 무위험수익률 명시(국고채 등), 소르티노·CAGR·손익비(payoff ratio) 추가, 승률과 손익비를 함께 제시(4장).

### [낮음/긍정] 올바르게 구현된 부분

### 3.1 [1순위] ATR 트레일링 / 샹들리에 청산 — ✅ 구현됨 (`type: "atr_trailing"`)

> 구현(`_atr_trailing_signals`): 진입=Donchian 돌파 `close > rolling_max(High, period).shift(1)`(당일 고가 제외, 미래참조 차단). 청산=샹들리에 `ChandelierExit = rolling_max(High, period) - k·ATR_atr_period` 를 종가가 하향 돌파. ATR 은 Wilder 평활(`alpha=1/atr_period`). 파라미터 `period`(기본 22), `atr_period`(기본 22), `k`(기본 3.0). 체결은 신호봉 종가(엔진 일관).

**개요**: 추세추종 진입에 **ATR(평균진폭) 기반 동적 손절/트레일링**을 결합. 기존 추세 전략(SMA/MACD/breakout)의 청산을 변동성 적응형으로 교체·보강.

**신호 수식**:
- `TR(t) = max(High-Low, |High-PrevClose|, |Low-PrevClose|)`
- `ATR_n = Wilder_EMA(TR, n)` (n=14, 22 통상)
- 롱 청산선(샹들리에): `ChandelierExit = max(High, n봉) - k·ATR_n` (k=3 통상)
- 진입은 Donchian 돌파(직전 N봉 최고 High 상향 돌파, `shift(1)`), 청산은 `close < ChandelierExit`

**근거**: 변동성에 비례한 손절폭으로 고변동 구간 조기 청산·저변동 구간 여유 확보. 현재 고정 % 손절(`stop_loss_pct`)의 한계를 보완.

**필요 데이터**: OHLC(고/저/종가). `get_ohlcv_frame` 로 공급.
**난이도**: 중. **적합 국면**: 추세장. **KRX 적합성**: 높음(상하한 ±30% 환경에서 변동성 적응 청산이 유효).

### 3.2 [2순위] 변동성 돌파 (래리 윌리엄스) — ✅ 구현됨 (`type: "volatility_breakout"`)

> 구현(`_volatility_breakout_signals`): `Target(t)=Open(t)+k·(High(t-1)-Low(t-1))`. 진입=`High(t) >= Target(t)`. 청산=**당일 종가 청산** 가정 — 동일 봉 동시신호 금지 규약 때문에 진입 봉의 다음 봉에서 청산(`entries.shift(1) & ~entries`). 전일 range 는 `shift(1)` 로 첫 봉 신호를 제거(미래참조 차단). 파라미터 `k`(기본 0.5). 일봉 백테스트 한계상 장중 목표 돌파를 봉 종료 후 평가하며 체결은 종가 기준.

**개요**: 당일 시가 + (전일 변동폭 × k) 돌파 시 매수. 일중 모멘텀을 노리는 한국에서 매우 대중적인 전략.

**신호 수식**:
- `Range(t-1) = High(t-1) - Low(t-1)`
- `TargetPrice(t) = Open(t) + k·Range(t-1)` (k=0.5 통상)
- 진입: 장중 `High(t) >= TargetPrice` (돌파 시 매수)
- 청산: 당일 종가 청산(데이트레이딩) 또는 익일 시가

**근거**: 전일 변동폭 대비 일정 비율 상향 돌파는 강한 일중 모멘텀 신호. Williams의 변동성 돌파.

**필요 데이터**: OHLC + 시가. **난이도**: 낮음(수식 단순). **적합 국면**: 추세개시·고변동. **KRX 적합성**: 매우 높음(국내 백테스팅 커뮤니티 표준 전략, 데이트레이딩 친화).
**주의**: 장중 체결 가정이라 **일봉 종가 체결 엔진과 충돌** — 일중 체결 로직(또는 보수적으로 익일 시가) 필요. 슬리피지 민감.

### 3.3 [3순위] 절대모멘텀 (타임시리즈 모멘텀) — ✅ 구현됨 (`type: "momentum"`)

> 구현: `lookback`봉 수익률 `mom = close/close.shift(lookback) - 1`의 0선 교차를 매매(진입=양전, 청산=음전). 월말 리밸런싱 대신 기존 크로스오버 이벤트 규약에 맞춰 부호 전환으로 신호화. 파라미터 `lookback`(기본 120).

**개요**: 과거 N개월 수익률이 양(+)이면 보유, 음(−)이면 현금. 가장 단순·강건한 모멘텀.

**신호 수식**:
- `Mom(t) = close(t)/close(t-L) - 1` (L = 12개월 ≈ 252봉 통상)
- 진입: `Mom(t) > 0`, 청산: `Mom(t) <= 0` (월말 리밸런싱)

**근거**: 자산 자체의 추세 지속성(Moskowitz·Antonacci). 하락장 회피로 MDD 개선. **close만으로 구현 가능 — 데이터 확장 불필요.**

**필요 데이터**: close. **난이도**: 낮음. **적합 국면**: 추세장(특히 하락장 방어). **KRX 적합성**: 높음(지수·개별주 모두).
**주의**: 월 1회 리밸런싱 구조라 현재 "교차 이벤트" 신호 프레임과 다름(리밸런싱 스케줄 도입 필요).

### 3.4 [4순위] 듀얼모멘텀 (상대+절대)

**개요**: 여러 자산 중 상대모멘텀 1위를 고르되, 절대모멘텀이 음이면 현금/안전자산. Antonacci.

**신호 수식**:
- 각 자산 `Mom_i = close_i(t)/close_i(t-L) - 1`
- 상대: `argmax_i Mom_i` 선택 → 그 자산의 `Mom > 0`이면 매수, 아니면 현금

**필요 데이터**: 다종목/지수 close + 안전자산. **난이도**: 중(다종목·리밸런싱 엔진). **적합 국면**: 추세장. **KRX 적합성**: 높음(코스피200 ETF·채권 ETF 조합).

### 3.5 [5순위] 평균회귀 z-score — ✅ 구현됨 (`type: "zscore"`)

> 구현: `z = (close - SMA_period)/std_period`. 진입=`z`가 `-entry` 상향 복귀, 청산=`z`가 0 상향 돌파. 파라미터 `period`(기본 20), `entry`(기본 2.0).

**개요**: 가격의 이동평균 대비 표준화 편차(z)가 임계 이하로 떨어지면 매수, 평균 회귀 시 청산.

**신호 수식**:
- `z(t) = (close(t) - SMA_n(t)) / σ_n(t)`
- 진입: `z(t)` 가 `-z_in`(예 -2) 상향 복귀, 청산: `z(t) >= 0`(평균 회귀)

**근거**: 볼린저밴드의 일반화(연속 z값으로 정량화). **close만으로 구현 가능.**
**필요 데이터**: close. **난이도**: 낮음. **적합 국면**: 횡보장. **KRX 적합성**: 중간(추세장 취약은 볼린저와 동일).

### 3.6 [6순위] 켈트너 채널 — ✅ 구현됨 (`type: "keltner"`)

> 구현(`_keltner_signals`): `Mid=EMA_ema_period(close)`(`adjust=False`), `Upper=Mid+mult·ATR_atr_period`. 진입=`close` 가 `Upper` 상향 돌파(`_cross_up`), 청산=`close` 가 `Mid` 하향 돌파(`_cross_down`). ATR 은 Wilder 평활. 파라미터 `ema_period`(기본 20), `atr_period`(기본 10), `mult`(기본 2.0). 당일 종가 기준 산출·종가 체결로 정합.

**개요**: EMA 중심선 ± ATR 배수 채널. 볼린저(표준편차)와 달리 ATR 사용 → 추세 추종형 돌파/청산.

**신호 수식**:
- `Mid = EMA_n(close)`, `Upper = Mid + m·ATR_p`, `Lower = Mid - m·ATR_p` (n=20, p=10, m=2)
- 진입: `close > Upper`(돌파 추종), 청산: `close < Mid`

**필요 데이터**: OHLC. **난이도**: 중. **적합 국면**: 추세장. **KRX 적합성**: 높음.

### 3.7 [7순위] 스토캐스틱 — ✅ 구현됨 (`type: "stochastic"`)

> 구현(`_stochastic_signals`): `%K=100·(close-LowestLow_k_period)/(HighestHigh_k_period-LowestLow_k_period)`(고저 동일 구간은 50 으로 0/0 방지), `%D=SMA(%K, d_period)`. 진입=과매도(`%K<lower`)에서 `%K`가 `%D` 상향 교차, 청산=과매수(`%K>upper`)에서 하향 교차. 파라미터 `k_period`(기본 14), `d_period`(기본 3), `lower`(기본 20), `upper`(기본 80, `lower<upper` 검증). 당일 봉 포함 rolling·종가 체결 정합.

**개요**: 최근 N봉 고저 범위 내 종가 위치(%K)와 그 이동평균(%D)으로 과매수/과매도 판단.

**신호 수식**:
- `%K = 100·(close - LowestLow_n)/(HighestHigh_n - LowestLow_n)`
- `%D = SMA(%K, d)`
- 진입: `%K`가 `%D`를 과매도(예 <20)에서 상향 교차, 청산: 과매수(>80)에서 하향 교차

**필요 데이터**: OHLC. **난이도**: 중. **적합 국면**: 횡보장. **KRX 적합성**: 중간(추세장 취약).

### 3.8 [8순위] 변동성 타게팅 (리스크 오버레이)

**개요**: 전략 자체가 아니라 **포지션 사이징 레이어**. 실현변동성이 목표(예 연 15%)를 넘으면 비중 축소.

**신호 수식**:
- `σ_realized = std(daily_return, n)·√252`
- `weight = min(1, σ_target / σ_realized)`

**근거**: 변동성 군집성 활용, MDD·테일리스크 완화. 현재 "전액 매수" 사이징(`risk.py`)의 핵심 개선.
**필요 데이터**: close(수익률). **난이도**: 중. **적합 국면**: 전구간(오버레이). **KRX 적합성**: 높음. **강력 추천** — 모든 전략에 공통 적용 가능.

### 3.9 [9순위] 거래량 동반 돌파 (OBV / VWAP)

**개요**: 가격 돌파를 거래량으로 확증해 거짓 돌파 필터링. breakout 전략의 보강.

**신호 수식(OBV 예)**:
- `OBV(t) = OBV(t-1) + sign(close(t)-close(t-1))·Volume(t)`
- 진입: 가격 신고가 돌파 AND OBV도 신고가(추세 확증)

**필요 데이터**: OHLCV(거래량). **난이도**: 중. **적합 국면**: 추세개시. **KRX 적합성**: 중간(저유동성 종목 거래량 노이즈).

### 3.10 [10순위] 이격도 (Disparity) — ✅ 구현됨 (`type: "disparity"`)

> 구현: `disp = 100·close/SMA_period`. 진입=`disp`가 `lower` 상향 복귀, 청산=`disp`가 `upper` 하향 이탈(RSI식 대칭 구조). 파라미터 `period`(기본 20), `lower`(기본 95), `upper`(기본 105).

**개요**: `이격도 = 100·close/SMA_n`. 100에서 멀어지면 평균회귀 베팅.
- 진입: 이격도 < 하한(예 95) 상향 복귀, 청산: 이격도 >= 100

**필요 데이터**: close. **난이도**: 낮음. **적합 국면**: 횡보장. **KRX 적합성**: 중간(국내 HTS 보편 지표).

### 3.11 [11순위] 섹터 로테이션

**개요**: 섹터(업종) ETF/지수 중 모멘텀 상위 섹터로 자금 순환.
**필요 데이터**: 다종목/업종지수. **난이도**: 높음(유니버스·리밸런싱). **적합 국면**: 전구간. **KRX 적합성**: 중간(KRX 업종지수·섹터 ETF 활용 가능하나 ETF 유동성 편차).

### 3.12 [12순위] 페어트레이딩 (통계적 차익)

**개요**: 공적분 관계 종목쌍의 스프레드 z-score 발산 시 한쪽 매수·한쪽 매도(시장중립).
**신호 수식**: `spread = log(P_A) - β·log(P_B)`, z-score 임계 발산 시 진입, 회귀 시 청산.
**필요 데이터**: 다종목 close, 공적분 추정. **난이도**: 높음(공매도 제약). **적합 국면**: 중립/횡보. **KRX 적합성**: 중간 — **개인 공매도 제약**으로 실거래 어려움(대주·공매도 가능 종목 한정). 현재 매수 전용 엔진과 구조적 충돌.

## 4. 공통 권고사항

### 4.1 백테스트 방법론

1. **워크포워드 분석(필수)**: In-sample에서 파라미터 최적화 → 인접 Out-of-sample 구간에서 검증을 롤링 반복. 단일 기간 최적화 결과는 신뢰하지 말 것. 파라미터 민감도(인접값 성과 안정성) 함께 점검.
2. **거래비용 모델(필수)**:
   - 매수 수수료: 위탁수수료(약 0.015% 내외, 증권사별 상이).
   - 매도 수수료: 위탁수수료 + **증권거래세 0.20%(2026년 코스피·코스닥 공통)**.
   - **왕복 실효비용 ≈ 0.23% 이상**을 기준선으로. 현재 기본 `fees=0.00015`는 비현실적 하한.
   - vectorbt 단일 `fees`로 매수/매도 비대칭을 못 살리므로, 매도 측 거래세를 별도 가산하거나 보수적으로 왕복비용을 `fees`에 합산.
3. **슬리피지(필수)**: 최소 1틱(호가단위) 또는 0.1~0.3% 고정. 체결가를 호가단위로 반올림. `Portfolio.from_signals`의 `slippage` 인자 사용.
4. **체결 가정 일치(필수)**: 신호 `t` → 체결 `t+1`(시가 또는 종가)로 1봉 지연 체결 표준화. 백테스트와 실시간 엔진(`risk.py` 경유)의 체결 가정을 동일하게. 당일 종가 신호의 당일 종가 체결은 비현실적임을 결과에 명시.
5. **시장 제약 반영**: 상·하한가 ±30% 도달 봉 체결 제한, 정규장 09:00~15:30, 결제 T+2(현금 회전), 거래정지/관리종목 제외.
6. **편향 점검**: 생존편향(상장폐지 종목 포함), 미래참조(지표·체결가), 데이터스누핑(전략 선택 자체의 과최적화).

### 4.2 리스크 관리

1. **포지션 사이징 개선**: 현재 "가용현금 전액 매수"(`risk.py` `qty = cash // price`)는 단일 종목 집중 리스크. **종목당 자본 비율 상한** + **변동성 타게팅**(3.8) 도입 권고.
2. **변동성 적응 손절**: 고정 % 손절을 **ATR 기반**(3.1)으로 보완. 백테스트 손절은 close-only라 장중 저가 손실을 과소평가 — OHLC 기반 평가로 개선.
3. **일일 손실 한도**: `check_daily_loss_limit`은 신규 진입만 차단. 한도 도달 시 **기존 포지션 청산 정책**을 별도 정의할지 결정 필요.
4. **상관 리스크**: EMA교차·MACD가 동일 EMA(12/26)를 공유해 신호 상관이 높음. 다중 전략 운용 시 분산 효과 측정 필요.

### 4.3 성과지표

현재 `total_return, mdd, sharpe, win_rate, num_trades`만 산출. 다음을 추가·보강 권고:

- **CAGR** `= (최종/초기)^(252/거래일수) - 1` — 기간 정규화 수익률.
- **소르티노** `= (연율수익 - rf) / 하방편차` — 하방위험만 고려, MDD 큰 전략 평가에 적합.
- **샤프**: 무위험수익률(rf, 국고채 등)을 명시(현재 vectorbt 기본 rf=0).
- **MDD + 회복기간(recovery)**: 최대낙폭과 회복 소요 기간 함께.
- **손익비(payoff ratio) = 평균이익/평균손실** + **기대값** — 승률만으로는 오해. 높은 승률이 큰 손실 한 번에 무너질 수 있음.
- **Calmar = CAGR/|MDD|**, 거래당 평균손익, 회전율(비용 영향 가늠).

지표 해석 시: 백테스트 샤프는 비용·슬리피지 반영 후 값이라야 의미 있음. 거래 수가 적으면(num_trades 소수) 통계적 신뢰도 낮음을 명시.
